# Keyword Strategy for Tech Roles

Sources: ResumeATS.net (2026), GeniusResume (2026), ATS Grade (2025), Engineering Resume Keywords Guide (2026)

Covers: How to identify, extract, and embed the right keywords for ATS and AI screening.

## Keyword Extraction Process

### Step-by-Step

1. **Copy job description** — Full text, not just requirements section
2. **Identify hard skills** — Technologies, languages, tools, certifications
3. **Identify soft skills** — Leadership, communication, methodologies
4. **Map to your skills** — Convert their terms to yours (exact match)
5. **Prioritize** — Required > Preferred > Nice-to-have
6. **Embed in resume** — Skills section + bullet points

### Extraction Example

**Job Description Excerpt:**
"Looking for a Senior Backend Engineer with experience in Python, Django, PostgreSQL, and AWS. Must have experience with CI/CD pipelines, RESTful API design, and microservices architecture. Agile methodology required."

**Extracted Keywords:**

| Priority | Keywords |
|----------|----------|
| P0 (Required) | Python, Django, PostgreSQL, AWS |
| P1 (Preferred) | CI/CD, REST APIs, microservices, Agile |
| P2 (Nice-to-have) | Senior, Backend, architecture |

## Keyword Placement

### Where to Put Keywords

| Location | Effectiveness |
|----------|---------------|
| Skills section | Highest — explicit keyword list |
| Bullet points (first 2 words) | High — early position matters |
| Job titles | High — exact title match |
| Summary | Medium — contextual placement |
| Project descriptions | Medium — detailed evidence |
| Education | Low — only if required |

### Density Guidelines

- **Skills section:** List 10-15 most relevant
- **Bullet points:** 1-2 keywords per bullet
- **Total keywords:** Match 70%+ of required + 50%+ of preferred

**Warning:** Too many keywords =稀释 (dilution). ATS weights each keyword; scatter-shot hurts relevance score.

## Common Keyword Mistakes

### What Doesn't Work

| Mistake | Why |
|---------|-----|
| Synonyms instead of exact terms | ATS does literal matching |
| Missing punctuation | "React.js" vs "React JS" vs "ReactJS" |
| Uncommon abbreviations | "K8s" may not match "Kubernetes" |
| Generic terms | "programming" not as strong as "Python" |
| Misaligned titles | "Lead Developer" doesn't match "Senior Engineer" |
| Missing variations | Include both "JavaScript" and "JS" if possible |

### What Works Better

| Wrong | Right |
|-------|-------|
| "code in Python" | "Python" (in skills), "Developed in Python" |
| "used CI/CD" | "CI/CD pipelines", "Jenkins", "GitLab CI" |
| "cloud experience" | "AWS", "GCP", "Azure", "EC2", "S3" |
| "team leadership" | "Led team of 5", "Managed 3 engineers" |
| "fast learner" | Demonstrate with projects |

| Microservices | Service-oriented, API gateway, containerization |
| REST | RESTful, REST API, web services |

| Skill | Tech-Specific Examples |
