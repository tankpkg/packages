# Bullet Point Formulas for Tech Resumes

Sources: Resumly (2026), Resume Worded (2025), CVCraft (2026), SoundCV (2026)

Covers: Action verbs, metric frameworks, and transformation examples for ATS-optimized bullets.

## The Bullet Formula

### Core Structure

```
[Action Verb] [What] [How] [Result (with metrics)]
```

### Breakdown

| Component | Purpose | Example |
|-----------|---------|---------|
| Action Verb | What you did | Developed |
| What | The deliverable | REST API |
| How | How you did it | using Node.js and Express |
| Result | The impact | serving 10K daily requests |

### Complete Example

- **Developed** REST API endpoints **using Node.js and Express** **reducing response time by 40%**

## Bullet Density

### How Many Bullets per Role

| Experience Level | Bullets per Role |
|-----------------|------------------|
| Entry (0-2 years) | 3-4 bullets per role |
| Mid (3-5 years) | 4-6 bullets per role |
| Senior (5+ years) | 5-7 bullets per role |

### Per Bullet Principles

- One primary achievement per bullet
- Combine related metrics in one bullet
- Lead with strongest impact
- Vary action verbs (don't repeat)

## Section-Specific Formulas

### Projects Section

```
[Project Name] | [Link]
- [What you built]: [tech stack] | [impact]
- [Key technical challenge]: [how you solved it]
- [User/scale metric]: [result]
```

Example:
```
E-Commerce API | GitHub
- Built RESTful API with Node.js and PostgreSQL handling 5K orders/day
- Implemented JWT auth reducing unauthorized access by 100%
- Optimized queries reducing response time from 800ms to 150ms
```

### Leadership Section

```
[Action] [team/context] resulting in [metric]
```

Example:
- "Led team of 4 engineers to rebuild payment system, reducing latency by 40%"
- "Mentored 2 junior developers, both promoted to mid-level within 12 months"
- "Established code review process improving code quality by 30%"

### Certifications

Simple format:
- "[Certification Name], [Issuer], [Year]"
- AWS Certified Solutions Architect, Amazon, 2024

## ATS Keyword Placement in Bullets

### First Position Matters Most

```
[Keyword] [rest of sentence]
```

| Less Effective | More Effective |
|----------------|---------------|
| "Built API using React" | "React: Built API handling 10K requests/day" |
| "Worked with AWS" | "AWS: Deployed services on EC2, S3, Lambda" |

### Embed Naturally

Don't: "Python Python Python"
Do: "Developed Python scripts, built Python API, wrote Python tests"

## Quick Reference: Common Phrases

| Instead of... | Use... |
|---------------|--------|
| "was responsible for" | "Developed", "Led", "Managed" |
| "helped with" | "Contributed to", "Assisted" |
| "worked on" | Specific action: "Built", "Implemented" |
| "good at" | Demonstrate with examples |
| "knowledge of" | Show applied knowledge |
| "experience with" | "Used", "Utilized", "Applied" |

## Checklist for Strong Bullets

- [ ] Starts with action verb
- [ ] Includes specific technology/term (keyword)
- [ ] Has measurable result
- [ ] Shows your specific contribution (not team)
- [ ] No passive voice
- [ ] No more than 2 sentences
- [ ] ATS would find keywords in first half
